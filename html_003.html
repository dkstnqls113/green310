시멘틱태그

HTML5의 가장 획기적인 태그
semantic - 의미론적인 이라는 사전적 의미를 갖는다.
시멘틱웹은 컴퓨터 프로그램이 코드를 읽고 의미를 인식할 수 있는 지능형 웹 언어를 표현

특징 - 각각의 의미하는 바가 정해져 있으므로 위치가 적절히 않은 곳에 쓰였을 경우 웹접근성 부분에서 오류도 파악하기도 한다.

시멘틱 종류

  • header 태그 - 머릿말, 페이지제목, 페이지 소개, 로고, 네비게이션(메뉴)를 담는다.
  • nav 태그 - 링크가 포함된 메뉴리스트들에 네비게이션
  • section 태그 - html 문서에 포함된 독립적으로 공간을 나누고자 할 때 사용하게 된다. 제목요소를 포함하고 있는 경우가 많다.
  • article 태그 - 섹션태그와 비슷한 성질을 가지고 있는 소단락 구간
    블로그포스트, 기사, 논편 등으로 활용
    제목태그는 h2~h6까지 담는다.
  • aside 태그 - 본문흐름에서 벗어나는 구조
  • footer태그 - 업체정보, 저작권정보, 사이트 맵 등 사이트의 주요알림 내용을 설정
  • figure태그 - 삽화, 다이어그램 등 문서의 주요 흐름의 독립적인 컨텐츠를 정의할 때 사용
  • figcaption태그 - figure 태그 내부에 첫번째 혹은 마지막 자식 태그로 사용되면서 figure 태그의 내용을 정의, 설명할 때 사용
